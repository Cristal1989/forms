<form>

Формы позволяют получать информацию посетителей веб страницы. Списки, рассылки, контактные данные, профиль пользователя, логин и регистрация, коментарии в блогах - все это реализуется с помощью форм и их элементов.

Формы - это "денежные страницы", благодоря которым интернет магазины продают свои товары, а онлайн сервисы получают оплату за предоставленные услуги. Компании измеряют успех своего веб сайта эффектиностью форм, потому что они отвечают на такие вопросы, как, сколько потенциальных клиентов наш веб сайт отправил отделу продаж и сколько людей подписались на наш продукт условно на прошлой неделе, поэтому формы постоянно рефакторятся и улучшаются.

Функциональная HTML форма состоит из 2х частей - интерфейс пользователя и сервер backend. Первое - это внешний вид формы (разметка и оформление), а второе - код обработки введенных данных (валидация, хранение данных, отправка элетронной почты и т.д).

Элемент <form>

Элемент form - это контейнер для группы связаных элементов формы, например форма отзыва, регистрации, олаты заказа в интернет магазине. Каждый из этих компонентов интерфейса состоит из группы интерактивных элементов, сгруппированых формой

index.html
<form name="signup_form" autocomplete="on" novalidate>
<label>
Email
<input type="email" name="email" />
</label>

<label>
Passwordbr
<input type="password" name="password" />
</label>

<button type="submit">Submit</button>
</form>

У элемента форм нет обязательных атрибутов, поэтому в примере несколько полезных необязательных.

  • name - уникальное на текузей веб страницк имя формы. Используется как на сервере так и на клиенте при работе с form. Может содержать символы английского алфавита в любом регистре, цифры, подчеркивания и тере. В имени нельзя использовать пробел.
  • autocomplite - определяет может ли браузер автоматически заполнять значения всех элементов формы. Имеет всего 2 значения off и on. Это поведение можно будет изменить для каждого элемента форм.
  • novalidate - атрибут-флаг, не имеет значения. Говорит браузеру не проверять валидность введенных данных при отправке формы. Если атрибуты не указаны, выполняется клиентская валидация.

При клике по <button type="submit"> или при нажатии клавиши Enter в любом поле формы, она «отправляется», что приводит к перезагрузке текущей страницы. Это поведение по умолчанию, которое можно будет изменить при помощи JavaScript.

<label>

Элемент label связывает описание (метку) с интерактивным элементом формы. Текст метки связан с элементом не только визуально, но и логически. Например screen reader огласит текст метки, когда пользователь поставит фокус на связаные элементы. При клике по метке связаный элемент автоматически получит эффект фокуса

Если элемент формы, например поле ввода вкладывается в label связь создается браузером автоматически. Клик по тексту mail или password поставит фокус в соответствующем поле ввода

index.html
<form>
<label>
Emailbr <input type="email" name="email" />
</label>

<label>
Password
<input type="password" name="password" />
</label>

<button type="submit">Submit</button>
</form>

Если элемент формы не вложен в label необходимо явно связать их через атрибут id связываемого элемента и атрибута for метки

index.html
<form>
<label for="user_email">Email</label>
<input type="email" name="email" id="user_email" />

<label for="user_password">Password</label>
<input type="password" name="password" id="user_password" />

<button type="submit">Submit</button>
</form>

Элемент input

Универсальный элемент для создания разнообразных полей ввода. Тип ввода определяется атрибутом type, значение которого по умолчанию текст - однострочное текстовое поле, применяющее любые символы

<input type="text" name="username" />

Концептуально тег input представляет запись в словаре, которая отправляется на сервер. Атрибут name определяет имя этой записи, а значение это то что ввел пользователь. Имя поля должно быть уникальным внутри формы. Значение атрибутов name так же используется при работе с формой в JavaScript

Атрибут autofocus

Поле ввода, которому задан этот атрибут, получит фокус при загрузке страницы и в нем можно сразу набирать текст. По умолчанию атрибут не установлен ни для какого поля.

Т.к фокус не может быть у нескольких элементов одновременно, атрибут задается полю, с которого пользователю стоит начать заполнять форму. Атрибут autofocus логический, т.е без значений, указывается только название.

Атрибут placeholder

Позволяет отображать текст-подсказку о том, какие данные необходимо ввести в поле, когда элемент input пуст. Этот атрибут можно использовать в любом элементе формы, где есть текстовое поле ввода.

Почта и пароль

Значение атрибута type влияет на тип поля, набор символов, который в него можно ввести и последующую валидацию значения. Например когда браузер смартвфона видит поле с типом email, он отображает пользователю специальную клавиатуру для ввода элетронной почтой с легко доступным символом @. Браузер так же может предоставить способ выбрать адрес электронной почты из адресной книги.

Если задать атрибуту type значение radio, input превратится в переключатель-радиокнопку (radio button). Радио кнопки всегда идут группами, что позволяет пользователю выбрать одно из множество предопределенных значений.

Выбор цвета

Выбор размера

В отличии от текстовых полей в радио кнопке нельзя воодить данные, поэтому каждый из них необходимо указать значение в атрибут value. Это значение, которое будет передано на сервер, когда пользователь отправит форму.

Атрибут checked указывает какой переключатель будет выбран (отмечен) по умолчанию. В группе радио кнопок в состоянии checked может быть только 1 элемент.

Check box - флажки

Флажки чекбоксы похожи на переключатели, но позволяют выбрать произвольное кол-во значений, т.е многие из многих. Чекбоксов может быть целая группа, например выбор хобби, либо только один, например принятие пользовательского соглашения при регистрации.

Ваше хобби

Числа

Обычное текстовое поле принимает буквально любые символы - цифры, буквы, пробелы, знаки пунктуауции и т.д. Для того чтобы разрешить вводить только числа с возможностью ограничить диапазон используется тип number. При вводе данных числовое поле браузер автоматически игнорирует все символы кроме цифр.

<label>
Age
<input type="number" name="age" value="0" min="18" max="120" />
</label>

Конечно же есть много чисел, которые не подходят для каждого типа числовых данных, например может быть необходимо запретить ввод слишком больших либо отрицательных чисел. Эта проблема решается с помощью атрибута min и max определяющий границы ввода

По умолчанию числовое поле принимает только целые числа. Это поведение можно изменить с помощью атрибута step, который указывает шаг изменения числа, по умолчание значение шага равно 1.

Телефонные номера

Тип tel создает поле ввода телефонных номеров, которые могут быть предоставлены в самых различных форматах. В настоящее време единственная польза от применения поля типа tel состоит отображение виртуальной клавиатуры для введение телефонных номеров в моб браузерах, которые содержат цифры но не буквы

<form>
<label>
Phone number
<input type="tel" name="phone_number" />
</label>

<button type="submit">Submit</button>
</form>

Выбраное значение 1

Браузеры не предоставляют никакой визуальной обратной информации об установленом значении. Это делается с помощью JS и доп разметки

Дата и время

В браузерах есть встроенных календарь, в котором пользователь может выбрать и/или время. Атрибуты min и max позволяют установить минимальные и максимальные даты, при условии использования правильного формата.

Из за различного внешнего вида в разных браузерах и сложности его изменения в CSS вместо нативного календаря чаще всего используют готовые JS библиотеки.

Элемент textarea

Создает многострочные текстовые поля для воода большого кол-ва текста. Например для обратной связи, коментарии, посты, уточнение деталей заказа

Атрибут rows устанавливает количество строк в высоту, cols колонок в ширину. На практике указывается только rows, а ширина жлемента контролируется через css.

По умолчанию элемент textarea можно растягивать по горизонтали и вертикали. Для того, чтобы контролировать изменение размера пользователем в css есть свойство resize

resize: both | horizontal | vertical | none

Элемент <select>

Выпадающее меню - альтернатива радио кнопкам, поскольку по умолчанию позволяет выбрать одно из многих вариантов. Элемент select это раскрывающее меню с атрибутом name, который содержит набор элементов - options c атрибутом value

Текст внутри элемента options отображается пользователю, а значение атрибута value это то, что будет использоваться при отправке формы.

По умолчанию выбран первый элемент options из списка. Это можно изменить задав необходимый опции атрибут-буль selected

Группировка опций

Элемент datalist

Автозаполнение - это прием с которым знакомы все пользователи. При вводе символов в тестовое поле поисковая система предлагает различные варианты автозаполнения. При наборе нового сообщения электронной почт почтовый клиент предлагает список получателей.

Элемент datalist создает список предопределенных значений, который можно выбрать при наборе в связаном текстовом поле. Этот список скрыт и становится доступным при получении инпутом фокуса и набора текста в нем. Список создаваемый элементом datalist связывается с текстовым полем через атрибут id. Его значение должно совпадать со значением атрибута list тега input.

Элементы options так же могут иметь атрибуты value. Это полезно когда текст опции и значение не совпадают. Например пользователь может не знать сокращение штатов в США, в выпадающем меню он увидит список штатов и их сокращения, а при выборе опции текстовое поле будет заполнено только значением опции

Группировка полей

Группировка связаных элементов делает формы более понятными для пользователей. Это так же помогает пользователю сосредоточится на небольших, упорядоченых группах полей, а не пытаться понять всю форму сразу. Группировка выполняется как визуально в интерфейсе так и логически в коде

Элементы <fieldset> и <legend>

Элемент fieldset это контейнер для группировки нескольких связаных элементов в форме, а вложеный legend выполняет роль заголовка группы. Связаные радиокнопки и чекбоксы всегда должны быть сгруппированы, другие типы полей группируются по необходимости.

Группировка с возможностью оформления

Элементы fieldset и legend имеют отличную симантику, но ограничены возможности оформления. В большинстве случаев для группировки элементов формы используется .div с атрибутами доступности role и aria-labeldby

<div role="group" aria-labelledby="contact-details-head">
<p id="contact-details-head">Enter your contact details</p>
Related elements
</div>

  • Атрибут role="group" указывает на то, что элементы внутри этого <div> являются частью группы.
  • Атрибут aria-labelledby="contact-details-head" содержит идентификатор элемента с описанием группы.
